# MotorCortex MVP 程序脚本设计和生产规范

> 版本：0.a
>
> 本文档用于指导 CAM 刀路后处理器或其他外部系统生成可被 MotorCortex MVP 直接加载执行的 Python 运动脚本。

---

## 1. 脚本本质

脚本是标准的 Python `.py` 文件，由系统通过 `exec()` 执行。运动命令（`LinearMove`、`JointMove`、`RigidBodyMove`、`FiveAxisMove`、`Dwell`）和组织命令（`BeginContour`、`EndContour`）作为全局函数注入到执行环境中，**无需 import**。脚本中可自由使用 Python 标准库（如 `math`）、变量、循环、条件判断等所有 Python 语法。

---

## 2. 坐标系约定

- 所有位置坐标单位为 **mm**，相对于 **WCS（工件坐标系）**
- WCS 和 TCP 在 `WorkStation/*.yaml` 中定义（YAML 中 position 单位为米，脚本中单位为 mm）
- 系统内部自动完成 WCS→世界坐标 和 TCP 偏移的变换，脚本只需关心 WCS 下的坐标
- 进给率单位：笛卡尔空间运动为 **mm/s**，关节空间运动为 **度/s**

---

## 3. 运动命令 API

### 3.1 FiveAxisMove — 五轴联动运动（CAM 刀路主要命令）

```python
FiveAxisMove(x, y, z, i, j, k, f)
```

| 参数 | 类型  | 单位 | 说明                           |
| ---- | ----- | ---- | ------------------------------ |
| x    | float | mm   | X 轴位置坐标                   |
| y    | float | mm   | Y 轴位置坐标                   |
| z    | float | mm   | Z 轴位置坐标                   |
| i    | float | —    | 刀具方向向量 X 分量（单位向量） |
| j    | float | —    | 刀具方向向量 Y 分量（单位向量） |
| k    | float | —    | 刀具方向向量 Z 分量（单位向量） |
| f    | float | mm/s | 进给率                         |

**说明：** 系统会自动归一化方向向量，但建议提供归一化后的值。位置线性插值 + 刀具方向球面线性插值（SLERP）。

### 3.2 LinearMove — 三轴直线运动

```python
LinearMove(x, y, z, f)
```

| 参数 | 类型  | 单位 | 说明         |
| ---- | ----- | ---- | ------------ |
| x    | float | mm   | X 轴位置坐标 |
| y    | float | mm   | Y 轴位置坐标 |
| z    | float | mm   | Z 轴位置坐标 |
| f    | float | mm/s | 进给率       |

### 3.3 RigidBodyMove — 六自由度刚体运动

```python
RigidBodyMove(x, y, z, i, j, k, ii, jj, kk, f)
```

| 参数 | 类型  | 单位 | 说明                                   |
| ---- | ----- | ---- | -------------------------------------- |
| x    | float | mm   | X 轴位置坐标                           |
| y    | float | mm   | Y 轴位置坐标                           |
| z    | float | mm   | Z 轴位置坐标                           |
| i    | float | —    | 工具 Z 轴方向向量 X 分量（第一姿态向量） |
| j    | float | —    | 工具 Z 轴方向向量 Y 分量               |
| k    | float | —    | 工具 Z 轴方向向量 Z 分量               |
| ii   | float | —    | 工具 X 轴方向向量 X 分量（第二姿态向量） |
| jj   | float | —    | 工具 X 轴方向向量 Y 分量               |
| kk   | float | —    | 工具 X 轴方向向量 Z 分量               |
| f    | float | mm/s | 进给率                                 |

**说明：** 两个姿态向量应正交，系统会自动归一化。适用于六轴机器人。

### 3.4 JointMove — 关节空间运动

```python
JointMove(j1, j2, j3, j4, j5, j6, j7=0.0, j8=0.0, j9=0.0, f=100.0)
```

| 参数   | 类型  | 单位 | 说明                     |
| ------ | ----- | ---- | ------------------------ |
| j1~j9  | float | 度   | 关节角度（最多支持9轴）   |
| f      | float | 度/s | 进给率                   |

### 3.5 Dwell — 停留/延迟

```python
Dwell(seconds)
```

| 参数    | 类型  | 单位 | 说明     |
| ------- | ----- | ---- | -------- |
| seconds | float | 秒   | 延迟时间 |

**说明：** 同步命令，Task Process 会等待 Plan Process 完成确认后才继续。

## 4. 组织命令

```python
BeginContour()   # 进入连续轨迹模式
EndContour()     # 退出连续轨迹模式
```

- 连续轨迹模式下，运动段之间不停止，实现平滑连续运动
- 适合 CAM 刀路的切削段
- 每个连续切削段用独立的 `BeginContour/EndContour` 对包裹
- 快速定位、抬刀/下刀等非切削段 **不要** 放在 Contour 内

---

## 5. 常用刀具方向参考

| 方向       | i     | j     | k      | 说明         |
| ---------- | ----- | ----- | ------ | ------------ |
| 垂直向下   | 0     | 0     | -1     | 最常用       |
| 垂直向上   | 0     | 0     | 1      | 反向         |
| 水平向右   | 1     | 0     | 0      | X 轴正方向   |
| 水平向左   | -1    | 0     | 0      | X 轴负方向   |
| 水平向前   | 0     | 1     | 0      | Y 轴正方向   |
| 45度倾斜   | 0.707 | 0     | -0.707 | 向 X 轴倾斜  |

## 6. 脚本结构模板

```python
import math

# ========== 参数定义 ==========
feed_rapid = 1000    # 快速移动进给率 mm/s
feed_cut = 500       # 切削进给率 mm/s
safe_z = 50          # 安全高度 mm

# ========== 快速移动到起始位置上方 ==========
FiveAxisMove(x=<start_x>, y=<start_y>, z=safe_z, i=0, j=0, k=-1, f=feed_rapid)

# ========== 下刀 ==========
FiveAxisMove(x=<start_x>, y=<start_y>, z=<start_z>, i=<i0>, j=<j0>, k=<k0>, f=feed_cut)

# ========== 连续切削轨迹 ==========
BeginContour()
for point in toolpath_points:
    FiveAxisMove(
        x=point.x, y=point.y, z=point.z,
        i=point.i, j=point.j, k=point.k,
        f=feed_cut
    )
EndContour()

# ========== 抬刀到安全高度 ==========
FiveAxisMove(x=<end_x>, y=<end_y>, z=safe_z, i=0, j=0, k=-1, f=feed_rapid)
```

## 7. 多 Pass / 多 Slice 刀路模板

```python
import math

feed_rapid = 1000
feed_cut = 500
safe_z = 50

# toolpath_passes: List[List[Point]]，每个 Pass 是一组连续刀路点
# 每个 Point 包含 x, y, z, i, j, k 属性

for pass_index, pass_points in enumerate(toolpath_passes):
    if not pass_points:
        continue

    first = pass_points[0]
    last = pass_points[-1]

    # 快速移动到 Pass 起点上方
    FiveAxisMove(x=first.x, y=first.y, z=safe_z, i=0, j=0, k=-1, f=feed_rapid)

    # 下刀
    FiveAxisMove(x=first.x, y=first.y, z=first.z, i=first.i, j=first.j, k=first.k, f=feed_cut)

    # 连续切削
    BeginContour()
    for pt in pass_points:
        FiveAxisMove(x=pt.x, y=pt.y, z=pt.z, i=pt.i, j=pt.j, k=pt.k, f=feed_cut)
    EndContour()

    # 抬刀
    FiveAxisMove(x=last.x, y=last.y, z=safe_z, i=0, j=0, k=-1, f=feed_rapid)

    Dwell(0.2)  # Pass 间短暂停留（可选）
```

## 8. 关键约束与注意事项

1. **刀具方向向量** `(i, j, k)` 应为单位向量（系统会自动归一化，但建议提供归一化后的值）
2. **快速定位与切削之间** 可用 `Dwell(0.5)` 分隔（可选，用于稳定）
3. **不同切削 Pass 之间的连接段**（Link/Retract）不要放在 `BeginContour/EndContour` 内
4. **每个连续切削段** 用独立的 `BeginContour/EndContour` 对包裹
5. **参数建议使用关键字参数形式**（`x=..., y=...`）提高可读性；位置参数形式也支持
6. 脚本中可使用 `print()` 输出调试信息，不影响运动执行
7. **指令内部结构**：每条命令被转换为 128 字节对齐的二进制指令，包含 9 个 double 位置槽位、进给率、命令类型、源代码行号和序列 ID
8. 脚本文件建议放置在 `scripts/examples/` 目录下，命名格式：`<机器人>_<模式>_<时长>.py`

## 9. 后处理器集成提示词

以下提示词可直接用于 CAM 后处理器项目中，指导 AI 或代码生成模块输出符合本规范的脚本：

> 你是一个 CAM 刀路后处理器的代码生成模块。你的任务是将计算好的刀路轨迹数据转换为 MotorCortex MVP 系统可直接加载执行的 Python 运动脚本（`.py` 文件）。
>
> **脚本格式规则：**
>
> 1. 脚本是纯 Python 文件，运动命令作为全局函数直接调用，无需 import。可自由使用 `math` 等标准库。
>
> 2. 所有位置坐标单位为 **mm**，相对于 **WCS（工件坐标系）**。进给率单位为 **mm/s**。
>
> 3. 可用的运动命令：
>    - `FiveAxisMove(x, y, z, i, j, k, f)` — 五轴联动，`(i,j,k)` 为刀具方向单位向量，`f` 为进给率 mm/s。这是 CAM 刀路的主要命令。
>    - `LinearMove(x, y, z, f)` — 三轴直线运动。
>    - `RigidBodyMove(x, y, z, i, j, k, ii, jj, kk, f)` — 六自由度刚体运动，`(i,j,k)` 为工具 Z 轴方向，`(ii,jj,kk)` 为工具 X 轴方向，两者应正交。
>    - `Dwell(seconds)` — 停留指定秒数（同步命令）。
>    - `JointMove(j1, j2, j3, j4, j5, j6, j7=0, j8=0, j9=0, f=100)` — 关节空间运动，角度单位为度，进给率为度/s。
>
> 4. 组织命令：
>    - `BeginContour()` / `EndContour()` — 连续轨迹模式，段间不停止，实现平滑连续运动。切削段应包裹在此对中。
>
> 5. 脚本结构模板：

>    # 连续切削轨迹
>    BeginContour()

>        FiveAxisMove(x=point.x, y=point.y, z=point.z,
>                     i=point.i, j=point.j, k=point.k, f=feed_cut)
>    EndContour()
>
>    # 抬刀到安全高度
>    FiveAxisMove(x=<end_x>, y=<end_y>, z=safe_z, i=0, j=0, k=-1, f=feed_rapid)
>    ```
>
> 6. 关键约束：
>    - 刀具方向向量 `(i,j,k)` 应为单位向量（系统会自动归一化，但建议提供归一化后的值）
